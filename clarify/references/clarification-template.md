# Clarification Template

Use this when the request is ambiguous enough to require explicit clarification before execution.

```markdown
## Requirement Clarification Summary

### Before
"[original request]"

### After
**Goal**: [the concrete outcome the user wants]
**Scope**: [what is included and excluded]
**Constraints**: [technical, product, policy, time, compatibility]
**Success Criteria**: [how to know the work is done correctly]

### Decisions Made
| Topic | Decision |
|-------|----------|
| [topic] | [decision] |
| [topic] | [decision] |

### Open Questions
- [only if any remain]
```

Prefer a short summary. Expand only when the task truly needs a richer spec.
